# Architecture Decision Records

Use this directory for choices that materially affect architecture, dependencies, persistence, security, or deployment. Do not create an ADR for routine implementation details.

Name records `NNNN-short-title.md`, starting with `0001`. Each record should contain: **Status**, **Context**, **Decision**, **Consequences**, and **Alternatives considered**. Status is one of `Proposed`, `Accepted`, `Superseded`, or `Rejected`.

Accepted records 0001–0008 cover the frontend architecture, IndexedDB wrapper,
routing, PWA generation, final Version 1 data rules, PWA security/update policy,
repository-scoped GitHub Pages deployment, and the integrated workout-flow extension.
